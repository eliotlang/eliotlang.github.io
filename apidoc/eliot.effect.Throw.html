Abilities

ability

Throw

ability Throw[E, F[_]]

The typed-error effect: a computation that may fail with an error value of type E.

The typed sibling of Abort, which discards the reason. A function that may fail declares {Throw[E]} and calls raise to stop with an E. raise returns a value of any type, so it drops straight into ordinary code, and the effect floats up to the caller. The success-or-error outcome is materialised only when the effect is discharged by runThrow, as an Either[E, A].

Programs can use String as E to just indicate an issue without being able to recover from distinguished error cases.

If recovering from different errors is expected, then programs should create their own error union type and raise that. Different error types compose in the effect set freely so using different error hierarchies comes at no cost to the user.

def parse(s: String): {Throw[String]} Tree = raise("not a tree")

Methods

def raise[A](err: E): F[A]

Fail with error err. Control does not return, so the result stands in for a value of any type A; runThrow reflects the failure into a Left.

ThrowCarrier

jvm
type ThrowCarrier[E, G[_], A]

The internal carrier that realises the Throw[E] effect: a ThrowCarrier[E, G, A] is a G-computation producing an Either[E, A].

Plumbing only — application code never names it. A function that may fail declares {Throw[E]} and discharges it with runThrow/catch; this carrier is what those operate on under the hood. Generic over the base carrier G, so it composes over IO, AbortCarrier[IO], and so on. Its representation (the Effect/Suspend/Throw instances) is supplied by the platform layer.

def

catch

stdlib
infix left def catch[E, G[_] ~ Effect, A](computation: ThrowCarrier[E, G, A], onError: E => A): {-Throw[E]} G[A]

Run a {Throw[E]} computation, recovering any raised error to a value of the same type as the success with onError. The everyday way to discharge Throw — reads like an exception handler and never exposes Either:

// Parse the configuration from a string
def parseConfiguration(config: String): {Throw[String]} Configuration = raise("Input not a valid configuration.")

// Recover with an empty configuration
def configOrDefault(config: String): Configuration = parseConfiguration(config) catch (_ -> emptyConfiguration)

On success it yields the computed value; on raise(err) it yields onError(err).

def

runThrow

abstract
def runThrow[E, G[_], A](obj: ThrowCarrier[E, G, A]): {-Throw[E]} G[Either[E, A]]

Discharge the Throw[E] effect, materialising it as an Either[E, A]: Right(a) if the computation completed, Left(err) if it failed. Any other effects remain in the carrier G. This is the single accessor of ThrowCarrier — the platform layer supplies its body (the generated ThrowCarrier data-field accessor, which cannot spell {-Throw[E]}; the annotation here rides onto it through the layer merge). Consume the result with foldEither, or reach for catch to recover in one step without ever naming Either.
