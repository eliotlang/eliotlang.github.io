Interval

compilerjvm
type Interval[T]

A closed interval [start, end] of values of a single numeric type T: a start and an end, both T. With Int's value range moved out of its type into the refinement channel (docs/bounds-as-refinements.md), both endpoints of a range share one type, so Interval needs only one type parameter.

An Interval is the general form of a numeric range: Interval[BigInteger] is the compile-time bound of an Int, while Interval[Int] is an ordinary runtime interval of machine integers. It closes under +/-/* (see the Numeric[Interval[T]] instance) purely by delegating to the Numeric and Compare of its endpoints — this is exactly why those abilities exist, so no interval arithmetic is written twice.

The type is abstract in the base (data is a chosen representation, which the base avoids); each platform layer supplies the concrete data Interval[T](start, end) and the method bodies. Like Int, the Numeric instance is split across layers: the base owns the platform-independent method signatures, and the platform layers supply the endpoint-wise bodies.

intervalJoin

def intervalJoin[T ~ Compare](a: Interval[T], b: Interval[T]): Interval[T]

The lattice join of two intervals — the least interval containing both, [min of the starts, max of the ends]. The plain-function form of the Meta[Interval[T]] instance's join (they compute the same thing): the compiler's auto-derived Meta[T$Meta] instances reach it transitively through their generated bodies, and a transitively- reached ability instance does not dispatch under the refinement channel's NbE evaluation — only natives re-fire — so a compound meta-structure's join must bottom out at this plain function (which itself bottoms at the endpoints' Compare natives), exactly as intervalAdd/… back the arithmetic transfer (docs/generic-refinement-merges.md).

withinByte

compiler
def withinByte(i: Interval[BigInteger]): Bool

True iff the interval i fits a signed byte / short / medium / long — the width-named where precondition predicates for an Int's tracked range: def useByte(x: Int): Int where withinByte(range(x)) demands, at each call, that the caller's computed range for x fits a signed byte, reducing to a Bool over the argument's concrete interval. They bake the width bounds (from BigInteger's fitsByte/…), so a where needs no BigInteger literals.
