Bool

abstract
type Bool

The Boolean truth values, true and false.

Bool is opaque — it exposes no pattern-match constructors, so it cannot be matched on. Branch on it with if..else or with fold, its two-armed eliminator; both are ordinary functions, there is no built-in conditional syntax.

fold

abstract
def fold[A](condition: Bool, whenTrue: A, whenFalse: A): A

The two-armed eliminator for Bool: yields whenTrue when condition holds, otherwise whenFalse.

Both arms must have the same type — a pure arm is not auto-lifted to match an effectful one — and fold adds nothing to the effect row: with effectful arms the result is simply the arms' shared carrier type, and only the selected arm is ever run. For everyday branching prefer if..else, which is built on fold and does lift a pure arm to match an effectful one.

The refinement of the result is the join of the two arms': which arm applies is decided only at runtime, so the compiler knows the result only as "one arm or the other" — e.g. an Int result's range covers both arms' ranges.

// Pure arms of one type: fold selects a value, adding nothing to the effect row.
def label(active: Bool): String = fold(active, "ON", "OFF")

// Effectful arms: both are {Console} Unit, and only the selected one runs.
def report(ok: Bool): {Console} Unit =
   fold(ok, printLine("all good"), printLine("check failed"))
def

if

stdlib
def if[T](condition: Bool, value: {Abort} T): {Abort} T

Conditional execution — the if of if..else: yields value when condition holds, otherwise short-circuits via the Abort effect.

Pair it with the infix else (in eliot.effect.Abort), which discharges the Abort with an alternative; else binds right-associatively, so chains spell naturally as if(..) else if(..) else ... Without an else the {Abort} floats up to the caller, making a bare if a guard inside an {Abort} function.

value may be a pure value (if(cond, "yes"), auto-lifted) or an effectful computation (if(cond, printLine("hi")), whose own effects join the row). Both branches of an if..else are built, but only the selected one is ever run, so this reads like an imperative if.

Since if curries just like all functions do, the second argument can appear after the first argument is applied separately, so if(cond, "yes") becomes if(conf) "yes", just like in an imperative language.

// A pure value chosen by a chain; else binds right-associatively.
def sign(n: Int): String = if(n > 0) "+" else if(n < 0) "-" else "0"

// Effectful branches: only the taken branch's printLine runs, and else discharges
// the {Abort}, so greet declares only {Console}.
def greet(known: Bool, name: String): {Console} Unit =
   if(known) printLine(name) else printLine("hello, stranger")

// No else: a guard — the {Abort} floats up to the caller.
def requirePositive(n: Int): {Abort} Int = if(n > 0) n

||

abstract
infix left at && def ||(a: Bool, b: Bool): Bool

Logical disjunction: true when at least one of a and b is true.

|| sits at the same precedence as && (unlike most languages, where && binds tighter), so a mixed chain groups left to right: a || b && c is (a || b) && c — parenthesise when you mean a || (b && c). Operands are ordinary values; there is no short-circuiting.
