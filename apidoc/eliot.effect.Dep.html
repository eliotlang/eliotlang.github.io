Dep

ability Dep[X, F[_]]

The dependency effect (a reader): read-only access to an environment value of type X that is supplied at the discharge site, not fixed globally.

A function that needs an X from the environment declares {Dep[X]} and calls dependency to read it; dependency is dispatched by the type expected at the call site, so distinct dependency types are distinct effects ({Dep[Database], Dep[Topic]}). The effect floats up to the caller unchanged, exactly like State or Throw — nothing about which X is provided is baked into the signature.

The value is injected only when the effect is discharged, by provide: it hands x to every dependency read of that type and yields the residual carrier.

// Lookup something from a database
def lookup(key: String): {Dep[Configuration], DBIO} Row = dependency.query("select ...")

// Provide the dependency in main
def main: IO[Unit] = printLine(
   lookup("42")
     .provide(Database("jdbc://app-db"))
     .rowText
)

DepCarrier

jvm
type DepCarrier[X, G[_], A]

The internal carrier that realises the Dep[X] effect: a DepCarrier[X, G, A] is a function from an environment X to a G-computation of the result — the reader transformer X -> G[A].

Plumbing only — application code never names it. A function that reads a dependency declares {Dep[X]} and discharges it with provide; this carrier is what that operates on under the hood. Generic over the base carrier G, so it composes over IO, another DepCarrier[Y, IO], and so on. Its representation (the Effect/Suspend/Dep instances) is supplied by the platform layer.

provide

stdlib
def provide[X, G[_], A](x: X, computation: DepCarrier[X, G, A]): G[A]

Discharge the Dep[X] effect by supplying the environment value x: every dependency read of type X in computation sees x, and the residual carrier G (still carrying any other effects) is returned.

// Some logic using two distinct dependencies
def logic: {Dep[Database], Dep[Topic]} Unit = ...

// Main providing both dependencies to run logic
def main: IO[Unit] =
   describe
     .provide(Database("jdbc://app-db"))
     .provide(Topic("events"))
