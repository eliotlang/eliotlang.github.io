Function

abstract
type Function[A, B]

A function from an argument of type A to a result of type B — the type of a lambda, and of any named value that takes one or more arguments. A named value with no parameters is an ordinary value of its result type, not a Function.

Functions of several arguments are curried: a three-argument function has type A => B => C, that is Function[A, Function[B, C]], so supplying one argument yields the function that takes the next. Spell function types with the infix => alias — A => B reads better than Function[A, B] and is the idiomatic form. A function is applied by naming its argument after it — the usual call syntax f(a), or bare juxtaposition f a (left-associative, so f a b is (f a) b); apply and the . operator are two further spellings of that same application.

.

stdlib
infix left below apply def .[A, B](a: A, f: Function[A, B]): B

Subject-first application: subject.f(rest...) is f(rest..., subject).

This gives Eliot an object-oriented reading with none of the object-oriented semantics: f stays an ordinary function, and the value to the left of the dot is supplied as its final, still-missing argument. Functions intended for chaining are therefore written with their subject as the last parameter. . is left-associative, so a chain reads in the order it runs — x.f(a).g(b) is g(b, f(a, x)) — instead of inside-out.

This is the idiomatic spelling for everyday code. It also restores the smart completion that point-free functional code usually loses: after typing list. an editor can offer exactly the functions whose final argument is a List, turning discovery into a menu.

// reads left-to-right, in the order it runs:
def result: Box[String] = Box("Hello").map(_ -> "Earth").as("World")
// ... desugars to the inside-out application:
//     as("World", map(_ -> "Earth", Box("Hello")))

=>

stdlib
infix right def =>(A: Type, B: Type): Type

The infix spelling of the function type: A => B is Function[A, B].

The idiomatic way to write function types; A => B is far more readable than Function[A, B], especially in higher-order signatures. It is right-associative, matching currying, so A => B => C is A => (B => C) — a function that returns a function.

def compose[A, B, C](f: B => C, g: A => B): A => C = a -> f(g(a))

apply

lang
def apply[A, B](f: Function[A, B], a: A): B

Apply a function to its argument — the named form of ordinary application: apply(f, a) is exactly f(a).

Application is normally written with the call syntax f(a) (bare juxtaposition f a also works), or subject-first with the . operator. apply names that same operation so it can be handled as a first-class value — passed to a higher-order function, or partially applied — wherever a plain Function is expected.

def bump(n: Int): Int = apply(increment, n)   // identical to increment(n)
