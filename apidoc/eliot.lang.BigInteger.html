type BigInteger

An arbitrary-precision integer with no bounds.

The type of integer literals and of the [MIN, MAX] bounds that parameterise Int, so it appears mainly at the type level when describing integer ranges.

def byteMin: BigInteger

The inclusive bounds of the standard signed machine-integer widths — 8-bit byte, 16-bit short, 32-bit medium, 64-bit long. These are the single definition of each boundary: the Byte/Short/Medium/Long aliases in Int and every platform's representation dispatch reference them rather than repeating the literals.

def fitsByte(min: BigInteger, max: BigInteger): Bool

Whether the range [min, max] fits in a signed byte / short / medium / long — the named width-containment predicates a platform's Int representation policy dispatches on (fitsIn specialised to each standard width).

fitsIn

def fitsIn(lo: BigInteger, hi: BigInteger, min: BigInteger, max: BigInteger): Bool

True iff the range [min, max] is fully contained in [lo, hi].

The range-containment test platform layers use to select an integer's representation and to gate range coercions. It reduces to a Bool once all four bounds are concrete.
