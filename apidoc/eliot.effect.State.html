moduleeliot.effect.State

State

ability State[S, F[_]]

The state effect: a computation that threads a state of type S, reading and replacing it.

A function that uses state declares {State[S]} and reads the current state as state or replaces it with putState. Neither wraps the result — the effect floats up to the caller. The result paired with the final state is materialised only when the effect is discharged by runStateToPair, as a Pair[A, S] (or runStateToValue / runStateToFinalState to keep only one side).

def tick: {State[Int]} Unit = putState(state + 1)

StateCarrier

jvm
type StateCarrier[S, G[_], A]

The internal carrier that realises the State[S] effect: a StateCarrier[S, G, A] maps an incoming state to a G-computation yielding the result paired with the new state.

Plumbing only — application code never names it. A function that threads state declares {State[S]} and discharges it with runStateToPair; this carrier is what that operates on under the hood. Generic over the base carrier G, so it composes over IO, AbortCarrier[IO], and so on. Its representation (the Effect/Suspend/State instances) is supplied by the platform layer.

def runStateCarrier[S, G[_], A](obj: StateCarrier[S, G, A]): {-State[S]} S => G[Pair[A, S]]

Unwrap a StateCarrier to its underlying state-transition function Function[S, G[Pair[A, S]]]. The single accessor of the carrier; the platform layer supplies the body (the generated StateCarrier data-field accessor, which cannot spell {-State[S]}; the annotation here rides onto it through the layer merge). Discharge the effect with runStateToPair, which threads an initial state through this function.

Discharge the State[S] effect, materialising it as a Pair[A, S]: the result paired with the final state, threaded from initial. Any other effects remain in the carrier G. Reach for runStateToValue or runStateToFinalState to keep only one side of the pair.

updateState

def updateState[S](f: S => S): {State[S]} Unit

Transform the current state by applying f to it: reads the state, replaces it with f(state), and yields unit. A derived convenience over state + putState — updateState(f) is exactly putState(f(state)) — so platform layers get it for free, without implementing a third State method.
