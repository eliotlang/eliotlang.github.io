WherePrecondition.els 
import eliot.effect.Console

// `where` proves x's range fits a byte —
// the compiler checks it at every call site.
def asByte(x: Int): Int where withinByte(range(x)) = x

def main: IO[Unit] =
  printLine(intToString(asByte(100)))

Small by construction

Emit only what you use

Generics and abilities resolve by monomorphization and dead-code elimination, so the compiler emits exactly the machinery your program needs — nothing general-purpose you didn't ask for.

Bounded & provable

Known before it runs

A value's range lives in the compiler's refinement channel. A where precondition proves it fits — checked at every call site, with zero runtime cost.

Effects in the type system

Say what a function may do

Effects are an unordered set in the signature — {Console, State, Throw[E]}. A function may perform only the effects it declares, and the compiler enforces it.

Simple to write

Direct style, no plumbing

Effectful steps sequence for you — no hand-written flatMap, monad transformers, or lifts. It reads like an ordinary modern language.

Real code, checked at compile time

The type system does the worrying

Every sample below is a real Eliot program from the compiler's example suite. Switch between them.

EffectsThrow.els 
import eliot.effect.Throw

// A parser that fails with a typed error on bad input.
def parseOk:  {Throw[String]} String = "parsed-value"
def parseBad: {Throw[String]} String = raise("malformed input")

// `catch` discharges {Throw[String]} in one step —
// no Either, no transformer, no ThrowCarrier in sight.
def main: IO[Unit] = {
  printLine(parseOk  catch (err -> err))
  printLine(parseBad catch (err -> err))
}
statically checked

raise and catch arrive from a single import. catch recovers a typed failure to a value of the same type, and once discharged the effect leaves the signature — the residual carrier becomes plain IO.

Match.els 
import eliot.effect.Console

data Maybe[A] = Nothing | Just(value: A)

def describe(m: Maybe[String]): String = m match {
  case Nothing -> "empty"
  case Just(v) -> v
}

def main: IO[Unit] = printLine(describe(Just("hello")))
exhaustive

Sum types are declared with data and taken apart with match. Constructors carry named fields, and the compiler checks that every case is handled.

Ranges.els 
import eliot.effect.Console
import eliot.lang.Numeric

// One integer type. A value's range is meta-information
// in the compiler, not a separate Int8 / Int16 / ...
def count: UnsignedByte = 7        // UnsignedByte is an alias for Int

def total: Int = count + count + count   // 21

def main: IO[Unit] = printLine(intToString(total))
zero runtime cost

No zoo of fixed-width integer types. Int is one type; its bounds ride along in the refinement channel and cost nothing at runtime — the range is a fact the compiler tracks, not a tag the program carries.

Effects in the type system

Effects live in the signature

Every function declares the capabilities it uses as an unordered set. The compiler reads that set, refuses any effect a function hasn't declared, and discharges each one explicitly at the edge of your program — no transformer stacks, no ordering baked in.

  • {Console} read and print lines
  • {Throw[E]} typed failures — raise / catch
  • {State[S]} threaded state, no mutable variables
  • {Dep[T]} injected dependencies — provide
  • {Abort} short-circuit — runAbort

EffectsMulti.els 
import eliot.effect.Dep
import eliot.effect.Log
import eliot.effect.Console

data Database(url: String)

// Three effects at once, in direct style.
def run: {Dep[Database], Log, Console} Unit = {
  log(dependency.url)
  printLine(readLine)
}

// Injected at the discharge site; residual carrier is IO.
def main: IO[Unit] =
  run.provide(Database("jdbc://app-db"))
$ ./mill examples.run jvm exe-jar \
examples/src/ -m EffectsMulti
